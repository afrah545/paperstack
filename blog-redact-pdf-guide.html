How to Redact a PDF for Free (Auto-Detect Aadhaar & PAN)

PaperStack Team · July 11, 2026 · 7 min read

Before you send a contract, ID copy, or bank statement to someone, there's usually an Aadhaar number, PAN number, or phone number you'd rather they not see. Redacting a PDF removes that content permanently — but most people redact PDFs wrong without realizing it, and manually hunting for every sensitive number on a long document is tedious. Here's how to do it properly, for free, with a tool that finds the sensitive bits for you.

🆔 Aadhaar 🧾 PAN 📱 Phone ✉️ Email

The mistake almost everyone makes

Opening a PDF and drawing a black rectangle over text with an annotation tool feels like redaction — but it usually isn't. That black box sits on a separate layer on top of the original text. The text underneath is still there. Anyone can select it, copy it, or extract it with basic tools. This has caused real, documented leaks in legal and government documents.

Real redaction means the underlying content is gone — not covered, not hidden behind a shape, actually removed from the file.

The bigger problem: finding what to redact

Even once you know how to redact properly, scanning a 10-page bank statement or KYC document line by line for every Aadhaar or account number is slow and easy to get wrong — miss one digit string and your ID is still exposed.

How to redact a PDF properly, step by step

  1. Open PaperStack's Redact PDF tool — no sign-up, nothing uploaded to a server.
  2. Upload your PDF. Every page renders as a thumbnail, and pages with likely sensitive data get a 🚩 flag automatically.

  6. Click "Done with this page" and repeat for any other flagged pages.

  8. Open the downloaded file and check every redacted area. Try selecting text where you redacted — if nothing is selectable there, it worked.

Why the result works

Once a box is confirmed — whether auto-detected or drawn manually — that page is converted into a flattened image before being placed back into the PDF. There's no separate text layer left underneath to extract; the pixels you see are all that exists on that page. The tradeoff: a redacted page is no longer searchable or copy-pasteable as text, since it's now an image. Pages you didn't touch stay exactly as they were.

Common questions

Is the auto-detection perfect?
No — pattern matching catches most standard-format Aadhaar, PAN, phone, and email formats, but it can miss unusual formatting or catch false positives (like a random 10-digit number that isn't actually a phone number). Always review every flagged page yourself before downloading.
Will this work on scanned documents or photos turned into PDFs?
Auto-detection needs a text layer to scan, so it won't catch anything on a pure scanned image. You can still manually draw redaction boxes on scanned pages the same way.
Can I undo a redaction after downloading?
No. Once the file is downloaded, that content is gone from that copy. Keep your original file separately if you might need the unredacted version later.
Does this remove metadata too, like author name or GPS data?
No — this tool only removes what's inside the boxes you draw or confirm on the page itself. For hidden metadata, that's a separate step worth checking before sharing.
Is my file uploaded anywhere, even for the auto-detection?
No. Text scanning, rendering, drawing, and rebuilding the PDF all happen in your browser using JavaScript — your file never touches a server.
