I Got Tired of "Free" PDF Tools That Weren't, So I Built My Own

10 min read · Updated June 2026

I needed to merge two PDFs for a job application back in early 2026. Simple task, right? First tool wanted my email. Second one let me merge, but when I tried to compress the same file five minutes later, it told me I'd hit my "2 free tasks per day." A third one did the job fine — and then I noticed it had quietly uploaded my resume to a server somewhere I knew nothing about.

That's the moment I decided to just build the thing myself. A few months later, PaperStack was live — 12 tools, everything running in your browser, nothing uploaded anywhere. No login wall after task #2. No "upgrade to Pro" popup mid-task.

I'm obviously not a neutral reviewer here — it's my tool. But I've used every competitor below for real work, not just for this post, so take my notes for what they are: a guy who got annoyed enough to write code about it.

What actually matters when picking a PDF tool

Forget the marketing pages. Here's what trips people up in practice:

  • Daily limits — does "free" mean free, or free-until-task-three?
  • File uploads — does your file leave your device, and if so, where does it go?
  • Account walls — can you actually use the tool without signing up?
  • Watermarks — is your output file clean, or stamped with someone's logo?
  • Tool coverage — do you need five different sites to merge, compress, and convert, or does one site do it all?

Where PaperStack actually wins

The honest answer: privacy and zero friction. Because everything runs client-side in your browser using JavaScript, your file genuinely never touches a server. If you're dealing with a payslip, an Aadhaar scan, a contract, or anything you'd rather not hand to a random company's cloud, that matters more than people think. You can disconnect your wifi after the page loads and the tools still work — that's the actual test for "client-side," not just a claim on a landing page.

The other win is no daily cap. Merge, then compress, then convert — back to back, as many times as you want, in one sitting. A lot of competitors will let you do the first one free and then nudge you toward a subscription for the second.

Where it doesn't win — being straight about it

I'm not going to pretend PaperStack beats Adobe at everything, because it doesn't:

  • Adobe Acrobat still has the best OCR and the most polished editing experience for genuinely complex documents — multi-column layouts, scanned legal contracts, that kind of thing. If you're doing heavy document work daily as part of your job, Acrobat's paid tier earns its cost.
  • iLovePDF and PDF24 have more tools and batch processing for some operations, like merging dozens of files at once. PaperStack covers the 12 tools most people actually reach for, but batch merge isn't built yet.
  • Smallpdf's interface is genuinely well-designed if you don't mind the 2-task daily cap and don't process sensitive files.

If you need OCR on a scanned 200-page PDF, go use Adobe or PDF24. If you're merging two files, compressing one, signing a form, or converting a resume to Word — and you'd rather not upload it anywhere — that's exactly what PaperStack is for.

So which should you actually use?

If you process a PDF once or twice a month and don't care about where the file goes — honestly, any of these will do the job. Use whichever interface you like.

If you do it weekly or more, the daily caps on Smallpdf and some iLovePDF tools will eventually annoy you. Either pay for a subscription, or switch to something without limits.

If you're dealing with anything sensitive — bank statements, ID scans, contracts, medical records — use a tool where the file doesn't leave your device. That's the whole reason PaperStack exists.

Common questions

Q.Is PaperStack actually free, or is there a catch?
No catch. No account, no daily cap, no premium tier waiting behind a paywall. It's free because it's not expensive to run — your browser is doing the work, not a server.
Q.Do I need to sign up?

Correct, for all 12 tools — processing happens locally using JavaScript in your browser. You can check this yourself: load the page, disconnect from the internet, and the tool still works.
